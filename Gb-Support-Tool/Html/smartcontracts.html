Smart Contracts

Brief History

Smart Contracts were first purposed in a white paper by "Nick Szabo" in 1994. He wanted to extend the functionality of electronic transaction methods, such as POS (point of sale), to the digital realm

Brief History

A Smart Contract is a computer program or a transaction protocol that is self-executed once the conditions are met. In the Crypto Industry, there are more and more Tokens/Coins that can execute smart contracts.

Below is a list of Coins/Tokens that can run smart contracts, and the corresponding programming language, this is not all the coins that can utilize smart contracts.

  • Ethereum
    • Solidity
    • Viper
  • Cardano
    • Haskell
  • EOS
    • C++
  • Tezos
    • Ligo
    • SmartPy
    • Morley
    • fi
    • Archetype
  • NEO
    • C#, VB.Net, F#
    • Java, Kotlin
    • Python
    • Go
    • JavaScript

On Ethereum Smart contracts are permissionless meaning anyone can write a smart contract. To write contracts you would need to learn a smart contract language (Solidity), as well as have enough ETH to launch the contract. Since deploying a contract is a transaction "Gas Fees" are required to pay. The Gas cost for deploying a contract is far higher than a normal ETH transfer.

t is recommended to test your contract on the test net before deploying to the main net.
